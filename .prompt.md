# Production Full-Stack Project Generator

You are an expert full-stack developer tasked with scaffolding a production-ready full-stack application.

## Input Parameters

Ask the user for these details (or use defaults if not provided):

- **Project Name**: (default: `MyAppProject`)
- **Description**: (default: `A production-level full-stack application`)
- **Backend Stack**: Node.js/Express, Python/Django, Go, etc. (default: `Node.js/Express`)
- **Frontend Stack**: React, Vue, Angular, Next.js, etc. (default: `React`)
- **Database**: MongoDB, PostgreSQL, MySQL, etc. (default: `PostgreSQL`)
- **Authentication**: JWT, OAuth2, Session-based (default: `JWT`)
- **Port Configuration**: Backend port (default: `5000`), Frontend port (default: `3000`)
- **Features**: Auth, Testing, CI/CD, Docker, Logging (default: `All`)

## Project Structure

Generate a complete production-ready structure with:

### Backend (`/server` or `/backend`)
```
├── src/
│   ├── config/          # Database, env, authentication configs
│   ├── controllers/      # Route controllers/handlers
│   ├── models/          # Database models/schemas
│   ├── routes/          # API route definitions
│   ├── middleware/       # Auth, error handling, logging
│   ├── services/        # Business logic
│   ├── utils/           # Helper functions
│   ├── tests/           # Unit & integration tests
│   └── index.js         # Entry point
├── .env.example         # Environment template
├── .eslintrc.json       # Code standards
├── docker-compose.yml   # Development database setup
├── package.json         # Dependencies & scripts
└── README.md            # Backend documentation
```

### Frontend (`/client`)
```
├── public/              # Static files
├── src/
│   ├── components/      # Reusable components
│   ├── pages/           # Page components
│   ├── services/        # API client services
│   ├── hooks/           # Custom React hooks
│   ├── utils/           # Helper functions
│   ├── context/         # State management
│   ├── __tests__/       # Component tests
│   ├── App.jsx
│   └── index.jsx
├── .env.example         # Environment template
├── .eslintrc.json       # Code standards
├── package.json         # Dependencies & scripts
└── README.md            # Frontend documentation
```

### Root Level
```
├── .git                 # Initialized git repo
├── .gitignore           # Git exclusions (node_modules, .env, builds, etc.)
├── docker-compose.yml   # Full stack compose (optional)
├── .github/
│   └── workflows/       # CI/CD pipelines (GitHub Actions)
├── docs/                # Project documentation
├── README.md            # Main project README
└── CONTRIBUTING.md      # Contribution guidelines
```

## Multi-File Generation

Create these essential files:

### Configuration Files
1. **package.json** (Backend & Frontend)
   - Production dependencies only
   - Dev dependencies clearly separated
   - Useful npm scripts
   - Correct version information

2. **.env.example**
   - All required environment variables
   - Safe default values (non-sensitive)
   - Clear comments

3. **.gitignore**
   - node_modules, build artifacts
   - Environment files, secrets
   - IDE and OS specific files
   - Logs and caches

4. **.eslintrc.json** & **.prettierrc**
   - Professional code standards
   - Consistent formatting rules

### Core Application Files
5. **Backend Entry Point** (src/index.js or main.py)
   - Server initialization
   - Database connection
   - Middleware setup
   - Error handling

6. **Database Connection** (src/config/database.js)
   - Connection pooling
   - Error handling
   - Connection logging

7. **Authentication Setup** (src/middleware/auth.js)
   - JWT verification
   - Token generation
   - Error responses

8. **API Routes** (src/routes/)
   - RESTful endpoints
   - Proper HTTP methods
   - Request validation

9. **Frontend App** (src/App.jsx)
   - Router setup
   - Context/state initialization
   - Layout structure

10. **API Client** (src/services/api.js)
    - Base API configuration
    - Request interceptors
    - Error handling

### Testing Files
11. **Backend Tests** (src/tests/)
    - Unit tests for utilities
    - Integration tests for API
    - Mocking strategies

12. **Frontend Tests** (src/__tests__/)
    - Component tests
    - Hook tests
    - Snapshot tests

### DevOps Files
13. **docker-compose.yml**
    - Backend service
    - Frontend service
    - Database service
    - Network configuration

14. **.github/workflows/ci.yml** (if GitHub)
    - Linting checks
    - Test execution
    - Build verification

15. **Dockerfile** (optional)
    - Multi-stage builds
    - Production-optimized
    - Security best practices

### Documentation
16. **README.md** (Root)
    - Project overview
    - Quick start guide
    - Tech stack
    - Contributing instructions

17. **Backend README.md**
    - API documentation
    - Environment setup
    - Database schema
    - Common issues

18. **Frontend README.md**
    - Component documentation
    - State management guide
    - Development workflow

## Best Practices to Include

- **Security**: Environment variables, input validation, CORS, HTTPS-ready
- **Error Handling**: Try-catch blocks, error middleware, proper status codes
- **Logging**: Console logs for development, file logs for production
- **Testing**: Test runners configured, sample tests provided
- **Performance**: Lazy loading, code splitting, database indexing setup
- **Scalability**: Modular structure, service separation, env-based config
- **Code Quality**: Linting configured, formatting rules set, comments for complex logic

## Output Format

1. Display file tree structure first
2. Generate complete, production-ready code for each file
3. Include comments explaining key sections
4. Provide setup instructions
5. List any manual configuration needed

## Notes

- All code should be production-ready (no TODO comments in critical sections)
- Include proper error handling and logging
- Database migrations/schemas should be included
- Security considerations should be commented
- Code should follow industry best practices
