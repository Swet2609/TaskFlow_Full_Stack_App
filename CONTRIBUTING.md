# Contributing to TaskFlow

We appreciate your interest in contributing to TaskFlow! This document provides guidelines and instructions for contributing.

## 🤝 Code of Conduct

- Be respectful and inclusive
- Focus on the code, not the person
- Help others learn and grow
- Report harassment or violations to maintainers

## 📋 Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/taskflow.git`
3. Create a branch: `git checkout -b feature/description`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push to your fork
8. Create a Pull Request

## 🔍 Pull Request Process

1. Update README.md with any new features
2. Add tests for new functionality
3. Ensure all tests pass: `npm test`
4. Lint your code: `npm run lint`
5. Provide a clear description of changes
6. Link related issues: "Closes #123"
7. Wait for review and respond to feedback

## 📝 Commit Messages

Format: `type(scope): description`

Examples:
- `feat(tasks): add priority filter`
- `fix(auth): resolve token validation bug`
- `docs(readme): update setup instructions`
- `test(tasks): add unit tests for taskService`

Types: feat, fix, docs, style, refactor, test, chore

## 🧪 Testing

- Write tests for all new features
- Maintain test coverage above 80%
- Run tests before pushing: `npm test`
- Test both success and error cases

## 📐 Code Style

- Follow ESLint configuration
- Use 2-space indentation
- Use meaningful variable names
- Add comments for complex logic
- Keep functions small and focused

## 🐛 Reporting Issues

- Check if issue already exists
- Provide clear title and description
- Include reproduction steps
- Attach screenshots if relevant
- Specify your environment (OS, Node version, etc.)

## ✨ Enhancement Suggestions

- Explain the use case
- Provide examples
- Consider implementation approach
- Discuss potential impact

Thank you for contributing! 🙏
